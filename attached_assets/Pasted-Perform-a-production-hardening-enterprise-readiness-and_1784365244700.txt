DEMO PAYMENT RULES

In demo mode:

* The user can choose a submission plan.
* The user can click a clearly labeled “Complete Demo Payment” button.
* The system creates a real persisted demo order record.
* The system creates a real persisted demo payment record.
* The submission proceeds through the normal review workflow.
* The submission status becomes pending review after the demo transaction succeeds.
* The user receives a demo confirmation number.
* No real money is charged.
* The interface must clearly state “Demo transaction — no money was charged.”
* Admin dashboards must label demo revenue separately from actual revenue.
* Demo payments must never be counted as real income.
* Demo payment records must be searchable and auditable.

For tips in demo mode:

* Users can select or enter a tip amount.
* The tip flow can be completed for demonstration.
* The system creates a persisted demo tip record.
* The creator dashboard may show the demonstration tip under “Demo Tips.”
* The tip must not be included in real earnings, payouts, taxes, revenue, or financial totals.
* Display “Demo tip — no money was transferred.”

* payment_status
* amount

Use values such as:

3. ROLE PERMISSION MATRIX

Document and enforce these responsibilities:

Master Admin:

* Full platform control
* Assign roles
* Configure legal and payment systems
* View all records
* Override decisions
* Manage licensing and white-label settings

Admin:

* Final submission approval
* Publish and unpublish content
* Manage users and content
* Handle DMCA and copyright actions
* Review payments and demo transactions
* Access platform analytics

Moderator:

* Review submissions for compliance
* Moderate comments, chats, profiles, and reports
* Approve submissions for admin final review
* Reject obvious spam or policy violations
* Escalate legal and copyright concerns
* Cannot publish final content
* Cannot issue final copyright strikes
* Cannot assign roles

Editor:

* Listen to songs and watch videos
* Create editorial playlists
* Recommend creators and content
* Suggest featured placement
* Promote discovery
* Cannot moderate users
* Cannot issue strikes
* Cannot access legal tools
* Cannot publish final content

Artist/Label:

* Manage their own profiles and content
* Submit media
* View only their own analytics
* Receive demonstration tips
* Cannot access staff tools

Listener:

* Stream, watch, follow, rate, comment, chat, message, save, report, block, and create playlists
* Cannot access submission or staff functions unless granted another approved role

Add automated authorization tests for every role.

4. REAL FILE UPLOAD SECURITY

For music, videos, artwork, avatars, banners, screenshots, reports, and attachments:

* Upload files from the user’s device
* Validate file extension
* Validate MIME type
* Validate file signature where possible
* Use server-generated random storage names
* Preserve the original filename only as metadata
* Enforce configurable size limits
* Enforce configurable duration limits where appropriate
* Reject executables and unsupported formats
* Prevent path traversal
* Prevent filename collisions
* Add upload scan status
* Add quarantine support
* Keep unpublished files private
* Make files public only after authorization or publication
* Prevent unauthorized access to private files
